# Contributing to Codex Limit Peek

Thanks for your interest in improving Codex Limit Peek ✨

## Before you start

- Read `AGENTS.md` for project principles and constraints.
- Keep changes local-first and avoid direct HTTP or credential access.
- Keep the menu bar UI compact (`percent | reset | weekly`) and avoid extra noise.
- Prefer small, focused PRs.

## Development setup

```bash
scripts/test.sh
scripts/build-app.sh
scripts/restart.sh
```

Contributor builds intentionally keep SwiftPM's `.build` cache. Validate the
end-user source installation separately with:

```bash
scripts/test-install.sh
```

## Code style

- Prefer SwiftUI/AppKit standard APIs.
- Keep behavior deterministic and avoid fake/mock fallback for quota values.
- Use clear, minimal labels in Chinese/English mix consistent with existing UI.

## Reporting bugs

- Reproduce with steps and expected/actual behavior.
- Mention macOS version, whether Codex CLI is signed in, and whether local `.codex/sessions` exists.

## Pull requests

- Keep diffs scoped.
- Add or update comments only when behavior is non-obvious.
- Make sure `scripts/test.sh` and `scripts/build-app.sh` pass locally.

## Release checklist (for maintainers)

1. Update version in `Info.plist` keys (`CFBundleShortVersionString`, `CFBundleVersion`) generated by `scripts/build-app.sh`.
2. Update `CHANGELOG.md`.
3. Run `scripts/test.sh` and `scripts/test-install.sh`.
4. Tag a release (for example: `v0.1.0`).
