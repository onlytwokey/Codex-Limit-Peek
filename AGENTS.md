# AGENTS.md

Guidance for AI agents working on Codex Limit Peek.

## Project

Codex Limit Peek is a macOS menu bar utility written with SwiftUI and AppKit. It reads current quota through the local Codex app-server, falls back to recent `~/.codex` records, and displays a compact menu bar status item plus a small popover panel.

## Principles

- Keep the app lightweight, local-first, and menu-bar focused.
- Do not add direct HTTP calls. Current quota reads must stay behind the local Codex CLI app-server boundary.
- Do not collect, upload, or expose user prompts, model replies, attachments, auth files, or other private session content.
- Treat quota values as inferred, not official.
- Preserve compact menu bar width.
- Prefer native macOS APIs and SwiftUI/AppKit over third-party dependencies.
- Keep UI changes glanceable and minimal; avoid full settings pages unless the feature genuinely needs one.

## Install

```sh
scripts/install.sh
```

Use `scripts/install.sh` for end-user deployment. It must keep SwiftPM build
artifacts in a system-temporary scratch directory and remove that directory on
normal exit and catchable termination signals. Do not replace it with a
repository-local release build for installation.

## Run

```sh

```

## UI Notes

- Menu bar text should remain short, currently `percent | reset | weekly`.
- The popover should prioritize 5-hour quota, reset countdown, and weekly quota.
- Buttons in the popover should stay compact and icon-based.
- Voice broadcast settings should stay in the More popover.

## Quota Parsing Notes

- Prefer `account/rateLimits/read` from a short-lived local Codex app-server process.
- Treat app-server failures, timeouts, missing fields, and logged-out state as normal fallback conditions.
- Local fallback records are read from SQLite headers or JSONL `payload.rate_limits`.
- Prefer aggregate Codex limits where `limit_id == "codex"`.
- Ignore model-specific limits such as `codex_bengalfox`.
- Reject local records whose source timestamp is more than 15 minutes old.
- For the current 5-hour window, prefer the highest observed `used_percent` among recent valid records to avoid transient `0%` records.
- Read file tails instead of entire JSONL files when possible; local session logs can be large.
